# PRD需求解析Skill

## 用途
将PRD产品需求文档解析为可理解、可执行的需求要点，用于指导后续代码开发。

## 解析维度

### 1. 功能模块提取
从PRD中提取所有功能模块，按端分类：
- PC端功能模块
- H5端功能模块

### 2. 页面结构分析
提取每个功能模块对应的页面：
- 页面名称
- 页面类型（列表/表单/详情）
- 核心功能点
- 涉及的数据表

### 3. API接口梳理
提取接口清单：
- 请求方法
- 请求路径
- 功能说明

### 4. 数据模型识别
提取相关数据表：
- 表名
- 核心字段
- 字段类型

## 输出格式

解析后输出以下结构：

```markdown
## 功能模块清单

### PC端
- 模块1
  - 页面：xxx
  - 类型：列表/表单/详情
  - API：xxx

### H5端
- 模块1
  - 页面：xxx
  - 类型：列表/表单/详情
  - API：xxx

## 核心数据表
- 表名：xxx
- 字段：xxx
```

## 使用方法

1. 读取PRD.md文档
2. 按功能模块逐章解析
3. 提取页面和API信息
4. 输出结构化需求要点
